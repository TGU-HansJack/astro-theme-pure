Astro帮助你[创作和展示你的内容](https://docs.astro.build/en/guides/content/)。你可以直接在Astro中使用Markdown/MDX编写博客文章，或从无头CMS获取内容。Astro让你围绕内容构建网站：你可以为页面添加布局，创建文章索引，并设置RSS订阅源让读者订阅。

## 编写内容

在Astro中，你可以通过多种方式创作内容：

- 在Markdown文件（`.md`或[替代扩展名](https://docs.astro.build/en/guides/markdown-content/)）中，专为编写富文本内容而设计。
- 在MDX（`.mdx`）或Markdoc（`.mdoc`）文件中使用[官方集成](https://docs.astro.build/en/guides/integrations-guide/)，可以在文档中包含组件和动态表达式。
- 使用[第三方内容管理系统(CMS)](https://docs.astro.build/en/guides/content/#headless-cms-authoring)，然后将内容拉取到`.astro`页面中。
- 其他选项（较少用于内容密集型页面）包括[`.astro`文件](https://docs.astro.build/en/basics/astro-pages/#astro-pages)和[`.html`文件](https://docs.astro.build/en/basics/astro-pages/#html-pages)。

对于本主题，默认支持`.md`和`.mdx`。

### Markdown创作

Markdown是一种方便的语法，用于编写带有基本格式和常见元素（如标题、列表和图片）的富文本。Astro内置支持项目中的Markdown文件。

在代码编辑器中创建并编写新的`.md`文件，或使用你最喜欢的Markdown编辑器引入现有文件。一些在线Markdown编辑器如[StackEdit](https://stackedit.io/)和[Dillinger](https://dillinger.io)甚至允许你编辑并与存储在GitHub上的Astro仓库同步你的工作。

让我们给你一个[在Astro中编写Markdown内容](https://docs.astro.build/en/guides/markdown-content/)的例子：

```markdown title="src/content/blog/first-article.md"
---
title: '第一篇文章' # (必需，最多60个字符)
description: '我喜欢写文章。' # (必需，10到160个字符)
publishDate: '2024-11-30 00:08:00' # (必需，日期)
tags:
  - Markdown # (也可以写成下一行的格式)
heroImage: { src: './thumbnail.jpg', alt: '针对我的文章的图片', color: '#B4C6DA' }
# thumbnail.jpg应该与文章在同一文件夹中
draft: false # (设置为true将仅在开发环境中显示)
language: 'English' # (你喜欢的字符串)
comment: true # (设置为false将禁用评论，即使你在site-config中启用了它)
---

## 这是一个标题

这是一个段落。
```

如果你有很多资源，可以为标题创建一个文件夹如`src/content/blog/first-article/`并将所有资源放在那里。当然，你的内容应该重命名为`index.md`或`index.mdx`并也包含在此文件夹中。

<Aside type='tip'>

如果你想为`heroImage`使用远程图片，请添加`inferSize: true`或为`heroImage`对象指定`width`和`height`。例如：

```yaml
heroImage:
  { src: 'https://img.tukuppt.com/ad_preview/00/15/09/5e715a320b68e.jpg!/fw/980', inferSize: true }
# 或指定宽度和高度
heroImage:
  { src: 'https://img.tukuppt.com/ad_preview/00/15/09/5e715a320b68e.jpg!/fw/980', width: 980, height: 551 }
```

</Aside>

太复杂了？简单例子只需要这些：

```markdown title="src/content/blog/simple-article.md"
---
title: 简单文章
description: 只是另一篇简单文章。
publishDate: 2024-07-26
---

我喜欢写简单文章。
```

<Aside type='tip'>
本主题有一个快速脚本为你创建新文章。只需在终端中运行`bun new <post-slug>`。
</Aside>

### MDX创作

这允许你在文本内容中包含UI元素，如横幅或交互式轮播。

直接在代码编辑器中编写和编辑`.mdx`文件，与项目文件一起。MDX文件是Astro中的[支持页面文件类型](https://docs.astro.build/en/basics/astro-pages/#supported-page-files)，也可以用作[内容集合条目](https://docs.astro.build/en/guides/content/#content-collections)。

对于集成组件，请查看[用户组件](/docs/integrations/components)和[高级组件](/docs/integrations/advanced)。

## 页面

Astro使用**基于文件的路由**来[根据项目`src/pages/`目录的文件布局生成你的构建URL](https://docs.astro.build/en/guides/routing/)。

### 路由

`.astro`[页面组件](https://docs.astro.build/en/basics/astro-pages/)以及`src/pages/`目录内的Markdown和MDX文件（`.md`，`.mdx`）**自动成为你网站上的页面**。每个页面的路由对应于其在`src/pages/`目录内的路径和文件名。

```diff
# 示例: 静态路由
src/pages/index.astro        -> mysite.com/
src/pages/about.astro        -> mysite.com/about
src/pages/about/index.astro  -> mysite.com/about
src/pages/about/me.astro     -> mysite.com/about/me
src/pages/posts/1.md         -> mysite.com/posts/1
```

<Aside type='tip'>

在Astro项目中没有需要维护的单独"路由配置"！当你向`src/pages/`目录添加文件时，会自动为创建新路由。在静态构建中，你可以使用[`build.format`](https://docs.astro.build/en/reference/configuration-reference/#buildformat)配置选项自定义文件输出格式。

</Aside>

### Astro页面

Astro页面使用`.astro`文件扩展名，并支持与[Astro组件](https://docs.astro.build/en/basics/astro-components/)相同的特性。

```astro title="src/pages/index.astro"
---

---

<html lang='en'>
  <head>
    <title>我的主页</title>
  </head>
  <body>
    <h1>欢迎来到我的网站！</h1>
  </body>
</html>
```

页面必须生成完整的HTML文档。如果没有明确包含，Astro将默认在`src/pages/`目录内的任何`.astro`组件中添加必要的`<!DOCTYPE html>`声明和`<head>`内容。你可以通过将组件标记为[部分](#page-partials)页面来选择退出此行为。

为避免在每个页面上重复相同的HTML元素，你可以将常见的`<head>`和`<body>`元素移动到你自己的[布局组件](/en/basics/layouts/)中。你可以使用任意数量的布局组件。

在本主题中，你可以使用`BaseLayout.astro`作为布局组件。

```astro title="src/pages/mypage.astro"
---
import PageLayout from '@/layouts/BaseLayout.astro' // [!code highlight]

const meta = {
  title: '我的页面',
  description: '我最喜欢的页面'
}
const highlightColor = '#44AEF6' // 可选
---

<PageLayout {meta} {highlightColor}>
  <!-- [!code highlight] -->
  <p>我的页面内容，包装在布局中！</p>
</PageLayout>
<!-- [!code highlight] -->
```

<Aside type='tip'>
  在Astro中阅读更多关于[布局组件](https://docs.astro.build/en/basics/layouts/)的信息。
</Aside>

### Markdown/MDX页面

虽然`.astro`页面具有一些灵活的特性和对内联/模块脚本友好，但`.md`和`.mdx`页面更适合内容导向的页面。

Astro还将`src/pages/`中的任何Markdown（`.md`）文件视为你最终网站上的页面。如果你安装了[MDX集成](https://docs.astro.build/en/guides/integrations-guide/mdx/#installation)，它也会以相同方式处理MDX（`.mdx`）文件。

<Aside type='caution'>

考虑为共享相似结构的相关Markdown文件目录（如博客文章或产品项目）创建[内容集合](/en/guides/content-collections/)，而不是页面。

</Aside>

Markdown文件可以使用特殊的`layout`前言属性来指定将包装其Markdown内容的[布局组件](https://docs.astro.build/en/basics/layouts/)在完整的`<html>...</html>`页面文档中。

本主题支持`.md`和`.mdx`文件的良好布局：

```markdown title="src/pages/terms/privacy.md"
---
layout: '@/layouts/IndividualPage.astro'

title: '隐私政策'
description: '生效日期: 2024-11-26'
language: 'En' # 可选
back: '/terms/list' # 可选，默认为'/'
heroImage: { src: './thumbnail.jpg' } # 将用于社交图片
---

## 隐私政策1

这是隐私政策的第一部分。
```

### HTML页面

具有`.html`文件扩展名的文件可以放置在`src/pages/`目录中并直接用作网站上的页面。请注意，一些关键的Astro功能在[HTML组件](https://docs.astro.build/en/basics/astro-components/#html-components)中不受支持。
