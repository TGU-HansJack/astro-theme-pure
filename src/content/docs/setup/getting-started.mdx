## 安装

两种安装方式。"模板"方式轻量简单，但难以更新；而"Fork"方式易于更新但需要一些git技能。

### 使用模板安装

<Aside type='tip'>
Astro有一篇很棒的文章教你设置项目：[安装和设置Astro #使用主题或启动模板](https://docs.astro.build/en/install-and-setup/#use-a-theme-or-starter-template) 
</Aside>

<Steps>

1. 安装主题

   在你的用户目录中执行以下命令来安装主题：
   
   <Tabs>
   <TabItem label='bun'>
   ```shell
   bun create astro@latest --template cworld1/astro-theme-pure
   ```
   <Aside type='tip'>
   对于Bun，如果安装缓慢，建议通过在项目根目录下创建`bunfig.toml`使用镜像配置：

   ```toml title="bunfig.toml"
   [install]
   registry = "<npm镜像站点>"
   ```

   有关其他PM镜像配置的详细信息，你可能需要查看它们的官方文档。
   </Aside>
   </TabItem>
   <TabItem label='pnpm'>
   ```shell
   pnpm create astro@latest --template cworld1/astro-theme-pure
   ```
   </TabItem>
   <TabItem label='yarn'>
   ```shell
   yarn create astro --template cworld1/astro-theme-pure
   ```
   </TabItem>
   <TabItem label='npm'>
   ```shell
   npm create astro@latest -- --template cworld1/astro-theme-pure
   ```
   </TabItem>
   </Tabs>

   默认情况下，此命令将使用模板仓库的主分支。要使用不同的分支名，请将其作为`--template`参数的一部分传递：`cworld1/astro-theme-pure#<branch>`。

2. 回答问题并按照CLI向导的指示操作。

3. 大功告成！

   你现在可以[启动Astro开发服务器](https://docs.astro.build/en/install-and-setup/#start-the-astro-dev-server)并查看项目实时预览，同时将其制作成你自己的！

</Steps>

### 使用Fork安装

你只需要在主题仓库中[点击fork按钮](https://github.com/cworld1/astro-theme-pure/fork)来创建你的项目；然后将fork的仓库克隆到你的本地机器。

```shell
git clone https://github.com/<你的用户名>/astro-theme-pure.git
```

然后，你可以启动Astro开发服务器并查看项目实时预览，同时将其制作成你自己的！

## 主题文件结构

主题文件结构如下：

- `public`：将复制到根目录的静态资源
- `src`：
  - `assets`：静态资源
  - `components`：主题中使用的组件，也包括用户类组件，如`Card`、`Collapse`、`Spoiler`等
  - `layouts`：基本站点布局
  - `pages`：页面如`404`、`about`、`blog`、`docs`、`index`等
  - `plugins`：主题中使用的扩展插件
  - `types`：Typescript类型定义
  - `utils`：工具
  - `site.config.ts`：主题配置文件
- `astro.config.mjs`：Astro配置文件
- `eslint.config.mjs`：ESLint配置文件
- `prettier.config.mjs`：Prettier配置文件
- `uno.config.ts`：UnoCSS配置文件
- `tsconfig.json`：Typescript配置文件
- `package.json`：包信息

## 简单设置


<Steps>

1. 移除文档文件

   - 移除`src/pages/docs`目录
   - 移除`src/site.config.ts`中的菜单声明：
   
   ```ts title="src/site.config.ts"
   export const theme: ThemeUserConfig = {
      // ...
      /** 配置站点的头部。 */
      header: {
         menu: [
            { title: '博客', link: '/blog' },
            { title: '文档', link: '/docs' }, // [!code --]
            // ...
         ],
      },
   }
   ```

   - 移除`src/content.config.ts`中的文档内容集合：

   ```ts title="src/content.config.ts"
   const docs = defineCollection({ // [!code --]
   loader: glob({ base: './src/content/docs', pattern: '**/*.{md,mdx}' }), // [!code --]
   schema: ({ image }) => // [!code --]
      z.object({ // [!code --]
         ... // [!code --]
      }) // [!code --]
   }) // [!code --]
   export const collections = { blog, docs } // [!code --]
   export const content = { blog } // [!code ++]
   ```

2. 移除`packages`目录（这将通过我们的NPM包导入）

3. 更改站点favicon。

   <Aside type='tip'>
   [Favicon.io](https://favicon.io/)是一个很好的工具，可以帮助你快速生成favicon。
   </Aside>

   用你自己的favicon替换`public/favicon/*`文件。

4. 替换你的头像图片。

   用你自己的头像图片替换`src/assets/avatar.png`文件。

5. 配置站点

   你可以在`src/site.config.ts`配置文件中配置你的项目：

   ```ts title="src/site.config.ts"
   export const theme: ThemeUserConfig = {
      author: 'CWorld',
      title: 'Astro Theme Pure',
      site: 'https://astro-pure.js.org',
      description: '求知若饥，虚心若愚',
      // ...
   }

   export const integ: IntegrationUserConfig = { /* ... */ }
   // ...
   ```

6. Typescript语法

   配置文件`site.config.ts`使用Typescript语法。如果你不熟悉TS语法，请先在这里[阅读](https://learnxinyminutes.com/docs/typescript/)。

</Steps>
